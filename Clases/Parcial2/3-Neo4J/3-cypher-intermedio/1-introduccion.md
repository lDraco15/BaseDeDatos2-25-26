# Pasando de escribir queries a diseñarlas

En la clase anterior trabajaste con patrones, filtros y recorridos. Ya puedes escribir consultas que funcionan. Pero ahora vamos a dar un paso más importante: dejar de escribir consultas “que salen” y empezar a diseñarlas con intención.

Esto marca una diferencia muy clara entre alguien que prueba cosas y alguien que entiende el modelo.

Antes de escribir nada, vamos a recuperar el estado del grafo.

```cypher
MATCH (n) RETURN n
```

Mira lo que tienes.

No avances todavía.

Este es tu punto de partida. Todo lo que hagas a partir de ahora debe tener sentido sobre este grafo.
